> 面向第一次接触这个仓库的人。目标：**看完能独立把 baseline 跑起来、看懂输入输出、并能换成自己的样例。**
> 配套阅读：《COT-TTS音频历史基线_架构与运作流程解析》（讲原理）、《TTS岗位简历与面试准备》（讲求职）。

## 第一部分：前置知识（看不懂代码？先补这些）

下面按"必须懂 / 最好懂 / 加分项"三档列出。**不用全部精通**，知道每个名词大概是什么、在这个项目里扮演什么角色即可。

### A. 必须懂（不懂会寸步难行）

| 知识点 | 你至少要知道 | 在本项目里的位置 |
|---|---|---|
| **Python 基础 + 命令行** | 会用终端 `cd`、跑 `python xxx.py --arg`、看报错 | 所有操作都靠命令行 |
| **Conda / 虚拟环境** | 会 `conda create/activate`，知道为什么要隔离环境 | 环境搭建 |
| **GPU / CUDA 基本概念** | 知道深度学习要显卡、`CUDA_VISIBLE_DEVICES` 选哪块卡 | 推理必须有 NVIDIA GPU |
| **什么是 TTS** | Text-To-Speech，文字→语音 | 整个项目的目标 |
| **什么是"模型权重/检查点(checkpoint)"** | 训练好的参数文件，要单独下载 | `model/` 目录 |

### B. 最好懂（懂了能看明白原理，面试也会问）

| 知识点 | 一句话理解 |
|---|---|
| **大语言模型(LLM)与自回归生成** | 像 GPT 一样"一个 token 接一个 token 往下写"，本项目用它来"写"出语音 token |
| **Token / 分词器(tokenizer)** | 把文本/音频切成离散编号；模型只认编号不认原始文字或波形 |
| **神经音频编解码器(Neural Audio Codec)** | 把一段 wav 压成一串离散整数 token，又能从 token 还原回 wav。本项目用的是 **BiCodec**（来自 Spark-TTS） |
| **离散语音 token 的两种角色** | **global token = 音色（像谁）**；**semantic token = 内容和语气（说什么、怎么说）**。这是全项目最关键的概念 |
| **链式思维(Chain-of-Thought, CoT)** | 让模型先"想"（输出情绪/动机/响度等推理），再"做"（生成语音）。本项目名字里的 CoT 就是这个 |
| **音色克隆 / zero-shot 说话人** | 给一段参考录音，让合成语音听起来像那个人 |

### C. 加分项（深入研究/改进时需要）

- **LoRA 微调**：在大模型上加一小撮可训练参数做适配，省显存。本 baseline 的模型就是 LoRA 微调出来的。
- **PyTorch / HuggingFace Transformers**：模型加载、`model.generate()` 推理 API。
- **采样超参**：`temperature`（随机性）、`top_p`（核采样）怎么影响生成。
- **分布式检查点(DCP)**：训练时的多卡保存格式，推理前要转成 HuggingFace 格式（本仓库会自动转）。
- **音频基础**：采样率(16kHz)、FLAC/WAV、响度(dBFS)、mel 频谱。

> **学习建议**：如果时间紧，先把 **B 档里的"两类 token"和"CoT"** 这两点弄懂，配合《架构解析》文档读一遍 `sample_case.raw.txt`，原理就通了一大半。

## 第三部分：四步把 baseline 跑起来

### 前提检查
- 一台带 **NVIDIA GPU** 的 Linux 机器（显存建议 ≥ 16GB；模型是 LLM 量级）。
- 已装 **conda**。
- 能访问 Google Drive 和 HuggingFace（下载模型/数据需要，可能要科学上网）。

---

### 步骤 1：拿到代码并进入目录
```bash
cd /path/to/ISCSLP2026-Challenge-baseline
```

### 步骤 2：搭建环境
README 提到一个 `env/spark_infer.yaml`，但**注意：当前仓库里并没有实际附带这个文件**。所以两条路：

**路线 A（推荐，手动建环境）**——按依赖手动装：
```bash
conda create -n spark_tts python=3.10 -y
conda activate spark_tts

# 核心依赖（按你的 CUDA 版本选对应 torch）
pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu121
pip install transformers safetensors accelerate
pip install soundfile numpy pyyaml einops omegaconf
pip install flash-attn --no-build-isolation   # 可选，加速；装不上见下方说明
```

**路线 B（如果你拿得到那个 yaml）**：
```bash
conda env create -f env/spark_infer.yaml
conda activate spark_audio_infer
```

> **flash-attn 装不上怎么办？** 脚本默认 `--attn_implementation flash_attention_2`。如果 flash-attn 编译失败，把启动脚本里这个参数改成 `--attn_implementation eager`（慢一点但能跑），或 `sdpa`。

> **缺包就装包**：跑的时候报 `ModuleNotFoundError: xxx` 就 `pip install xxx`。常见还会用到 `librosa`、`scipy`。（注意 `sklearn` 不用装，代码里有轻量桩替代。）

### 步骤 3：下载模型（关键，3 个包都要）
按 README 的链接，把三个模型包下到 `model/` 下，最终长这样：
```
model/
├── Spark-TTS/                          BiCodec 编解码器 + wav2vec2（音频↔token 必需）
├── cot_tts_audio_history_baseline/     音频历史基线模型
│   ├── checkpoints/global_step_37000/  ← 检查点
│   ├── model_assets/                   ← tokenizer 等资源
│   └── veomni_cli.yaml                 ← LoRA 配置
└── cot_tts_text_history_baseline/      文本历史基线模型（跑文本基线才需要）
    └── checkpoints/global_step_24500/
```
- 跑**音频历史基线**：必须有 `Spark-TTS/` + `cot_tts_audio_history_baseline/`。
- 三个下载链接都在 README 第 3 节。

### 步骤 4：一键运行
```bash
CUDA_VISIBLE_DEVICES=0 bash infer/run_cot_tts_inference.sh
```
- `CUDA_VISIBLE_DEVICES=0` 指定用第 0 块 GPU；不设的话脚本会自动挑当前最空闲的卡。
- 跑完去 `infer/results/` 看结果。

**成功标志**：终端打印 `[case sample_case] ... wav=infer/results/sample_case.wav`，且 `results/` 下生成了 `sample_case.wav`。

## 第四部分：看懂输出

跑完后 `infer/results/sample_case.*` 会有一堆文件，重点看这几个：

| 文件 | 看什么 |
|---|---|
| **`sample_case.wav`** | 最终合成语音，直接放来听 |
| **`sample_case.raw.txt`** | 模型完整原始输出。**新手必看**——能直观看到"历史转写 → CoT 推理 → 语音 token"三段结构 |
| `sample_case.cot.txt` | 单独抠出的 CoT 推理（情绪/动机/响度等 7 个字段） |
| `sample_case.history_transcript.txt` | 模型对历史音频的转写 |
| `sample_case.meta.json` | 元数据：用了多少 token、音色来自 ref 还是模型生成、选了哪条候选 |

打开 `raw.txt` 你会看到类似：
```
<under_start>Just listen. I am the queen...[自信坚定]<under_end>   ← 历史转写
<cot_start> <Act>:直接表达不满 ... <Emotion>:情绪从平静转向不满 ...   ← CoT 推理
[Summary] 带着失望又生气的情绪说了'It is so unfair.' <cot_end>
<audio_tar_start> <|bicodec_global_..|> <|bicodec_semantic_..|> <audio_tar_end>  ← 语音token
```
这就是 CoT-TTS 的精髓：**先理解语境 → 再推理该怎么说 → 最后"说"出来。**

## 第五部分：换成你自己的样例

最简单的方式——直接替换 `infer/cases/sample_case/` 里的文件：

1. **`target.txt`**：改成你想合成的句子（一行文字即可）。
2. **`history.wav`**：换成你的对话历史录音（前文语境）。
3. **`reference.wav`**：换成你想要的音色的录音（输出会像这个人）。
4. 重新跑 `bash infer/run_cot_tts_inference.sh`。

进阶——不覆盖样例，新建一个 case 目录并用命令行指定：
```bash
# 在 infer/cases/ 下新建 my_case/，放入 history.wav / reference.wav / target.txt
CUDA_VISIBLE_DEVICES=0 python infer/cot_tts_inference.py \
  --case_names my_case \
  --checkpoint_path model/cot_tts_audio_history_baseline/checkpoints/global_step_37000 \
  --spark_model_dir model/Spark-TTS \
  --model_architecture lora \
  --use_ref_global_tokens \
  --do_sample --temperature 0.6 --top_p 0.8
```

**音频小贴士**：参考音频尽量干净、单说话人、几秒到十几秒；脚本会自动去首尾静音。历史音频代表"语境"，它的情绪语气会影响模型的表演判断。

## 第六部分：常用参数速查

| 参数 | 作用 | 新手建议 |
|---|---|---|
| `--case_names` | 要跑哪些 case（可多个） | 默认 `sample_case` |
| `--use_ref_global_tokens` | 输出音色用参考音频（而非模型生成） | **建议开**，音色更稳 |
| `--history_mode` | 历史音频编码方式：`full`/`semantic`/`full_first_then_semantic` | 默认 `full` |
| `--do_sample` `--temperature` `--top_p` | 采样随机性 | 想更稳定就调低 temperature；想更多样就调高 |
| `--num_candidates` `--rerank_metric` | 生成 N 条候选并重排选优(`mel_similarity`) | 追求音色一致可设 `--num_candidates 4 --rerank_metric mel_similarity` |
| `--max_new_tokens` | 最长生成长度 | 默认 2000 够用 |
| `--device` | 用哪块卡，如 `cuda:0` | 配合 `CUDA_VISIBLE_DEVICES` |

## 第七部分：常见报错与排查

| 现象 | 可能原因 / 解决 |
|---|---|
| `checkpoints dir not found` / `model_assets dir not found` | 模型没下全或路径不对，核对第三部分目录结构 |
| `flash_attn` 相关报错 | 把 `--attn_implementation` 改成 `eager` 或 `sdpa` |
| `CUDA out of memory` | 换显存更大的卡；或确认没有别的进程占卡；`num_candidates` 调回 1 |
| `ModuleNotFoundError: xxx` | `pip install xxx`（sklearn 除外，有桩） |
| 输出 `*.failed.*` 文件、提示 `no bicodec semantic tokens` | 这次生成没解析到语音 token；重跑、或调采样参数/加 `min_new_tokens` |
| 没有 GPU | 代码会回退 CPU，但 LLM 推理在 CPU 上会极慢，不建议 |
| 下载链接打不开 | Google Drive / HuggingFace 需要科学上网 |

## 第八部分：推荐学习路径（新手 7 天计划）

1. **Day 1**：搭环境 + 跑通 sample_case，听一遍输出 wav。
2. **Day 2**：精读 `sample_case.raw.txt` 和 `meta.json`，对照《架构解析》文档理解三段结构。
3. **Day 3**：换自己的 target/reference/history 跑几个例子，感受参数影响。
4. **Day 4**：读 `infer/cot_tts_inference.py` 的 `build_prompt` 和 `infer_case` 主流程。
5. **Day 5**：搞懂 BiCodec 两类 token（读 `sparktts/models/audio_tokenizer.py`）。
6. **Day 6**：玩参数——扫 temperature/top_p、试 `num_candidates + mel_similarity` 重排，记录差异。
7. **Day 7**：对比跑一下 text-history 基线，理解两条基线的区别。

跟完这 7 天，你就从"会跑"进阶到"会讲、会改"了。
