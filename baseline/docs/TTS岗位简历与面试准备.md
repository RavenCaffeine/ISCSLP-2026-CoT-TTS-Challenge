## 1. 这个项目为什么对口 TTS 岗位

它几乎覆盖了当下语音生成岗位最看重的全部关键词：

- **LLM-as-TTS-backbone**：用大语言模型自回归生成离散语音 token，是 2024–2025 年 TTS 的主流范式（VALL-E / SparkTTS / CosyVoice 一脉）。
- **神经音频编解码器（Neural Audio Codec）**：BiCodec 的 global/semantic 双码本、FSQ/残差量化、Wav2Vec2 特征、说话人编码器。
- **音色克隆 / zero-shot 说话人**：用参考音频 global token 控制输出音色。
- **可控/富表现力合成 + CoT**：情绪、响度、表现强度等结构化控制。
- **LoRA 高效微调、分布式检查点(DCP→HF)、推理工程**：工程落地能力。

这些正是面试会问的东西。你只要能把第 2 节那几条讲透，就已经超过大量只会调 API 的候选人。

## 2. 你需要真正搞懂的 6 个点（面试必问）

在写简历之前，确保你能用自己的话讲清楚以下内容（详见配套的《架构与运作流程解析》文档）：

1. **整体数据流**：历史音频 + 目标文本 + 参考音频 → 编码成 token → 拼 prompt → LLM 生成 CoT + 语音 token → BiCodec 解码成 wav。
2. **BiCodec 两类 token**：`global`=音色身份（取前 32 个），`semantic`=内容/韵律。为什么解耦、解耦带来什么好处（音色可替换）。
3. **Prompt 协议**：`<task_start>COT-TTS` / `<history>` / `<target>` / `<output><cot_start>` 的结构，以及为什么 prompt 以 `<cot_start>` 结尾（引导模型先推理再合成）。
4. **CoT 的 7 个字段**：Act/Scene/Motivation/Goal/Emotion/Duration/Loudness+Expressive Intensity/Summary，它们如何引导表现力。
5. **音色来源选择**：`use_ref_global_tokens` 时输出音色来自参考音频，模型只生成 semantic——这是 zero-shot 音色克隆的实现方式。
6. **推理工程兜底**：DCP→HF 自动转换、LoRA 自动探测、三级 token 解析、多候选 mel_similarity 重排。

> 自测：如果有人问"为什么要先生成 CoT 再生成语音，而不是直接出语音？"你能答上来吗？（参考答案：把"该用什么情绪/动机/强度说"显式建模出来作为条件，提升语境契合度和可解释性，也便于可控生成与调试。）

## 5. 面试高频问答准备（带参考答案要点）

**Q1：介绍一下这个项目 / 它的整体架构？**
> 三组件：BiCodec 音频编解码器 + LoRA 微调的 LLM 主干 + 推理编排。数据流照第 2 节第 1 点讲。一分钟讲完主线，再等追问。

**Q2：BiCodec 的 global 和 semantic token 有什么区别？为什么要分开？**
> global 编码音色身份（who），semantic 编码内容韵律（what & how）。解耦后可以"固定音色、只换内容"，从而用参考音频做 zero-shot 音色克隆，输出音色稳定可控。

**Q3：为什么用 LLM 做 TTS？相比传统 TTS 优势？**
> 把文本、音色、内容统一成离散 token 用一个自回归模型建模，天然支持 in-context 条件（历史、参考音色、CoT），可控性和语境感知强；劣势是推理慢、可能有重复/漏字，需要停止 token 和重排兜底。

**Q4：CoT 在这里起什么作用？去掉会怎样？**
> CoT 让模型先显式推理情绪/动机/响度/表现强度，再据此合成，提升语境契合与可解释性。去掉 CoT 预计语境表现力下降——这正好是你可以做消融实验回答的（见第 4 节）。

**Q5：怎么保证输出音色像参考人？**
> `use_ref_global_tokens` 让解码使用参考音频的 global token，模型只生成 semantic；另有 mel_similarity 多候选重排进一步选最像的一条。

**Q6：LoRA 是什么？为什么用它？**
> 在冻结的大基座上注入低秩适配矩阵（rank-8）只训练少量参数，省显存/存储、便于多任务多检查点切换。这里 target 是 attention 和 MLP 的投影层。

**Q7（陷阱）：模型是你训练的吗？**
> 诚实回答：模型权重是官方基线，我做的是复现、源码级分析和 [你的改进]。——千万别吹训练。

**Q8：这个系统有什么局限 / 你会怎么改进？**
> 自回归推理慢、长文本稳定性、CoT 与最终语音可能不一致、评测主观。改进：非自回归/流式解码、CoT 一致性约束、更强的客观评测与重排——把第 4 节的想法讲出来。
