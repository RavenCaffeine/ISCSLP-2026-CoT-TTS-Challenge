# 科研新人从 0 到 1 完成第一篇论文（以 CoT-TTS / ICASSP 为例）

> 写给第一次写论文的人。目标不是教你"高深技巧"，而是给你一张**完整的流程地图 + 每一步具体怎么做**，让你不至于在某个环节卡死。
> 全程用你手上的 CoT-TTS 工作做例子。配套：《投递ICASSP备战指南》《Track1任务文档》。

---

## 0. 先纠正三个新手最常见的误解

1. **"要有惊天动地的创新才能发论文"** —— 错。一篇合格的会议论文 = **一个清晰的小问题 + 一个具体的解法/发现 + 扎实的实验证明**。你的"语音-推理一致性度量"就足够。
2. **"先把所有实验做完，最后再写"** —— 错。**论文的骨架（尤其是贡献点和实验表格）要先想清楚**，它会反过来指导你做哪些实验。边想边写边做。
3. **"写作是最后润色的事"** —— 错。写作是**思考本身**。很多逻辑漏洞、缺的实验，都是在写的时候才暴露出来的。所以要早写。

> 核心心法：**一篇论文 = 回答一个问题。** 从头到尾，你都要能用一句话说清"我在回答什么问题，答案是什么"。

---

## 1. 论文的"解剖结构"：先看懂目标是什么

一篇 ICASSP 论文本质是一条**逻辑链**，每一节都在为它服务：

```
有个问题 (Intro)
  → 别人没很好解决 (Related Work)
    → 我提出这个方法 (Method)
      → 实验证明它有效 (Experiments)
        → 所以结论是… (Conclusion)
```

新人最容易写成"我做了 A，又做了 B，还做了 C"的流水账。**永远记住：不是记录你干了什么，而是论证一个结论。** 没有服务于核心结论的内容，再辛苦也要删。

---

## 2. 从 0 到 1 的七个阶段（总览）

| 阶段 | 做什么 | 产出 | 你现在的状态 |
|---|---|---|---|
| ① 找问题 | 锁定一个具体、可验证的贡献点 | 一句话研究问题 | ✅ 已有候选(一致性度量/CoT消融) |
| ② 读文献 | 搞清楚别人做到哪、你的位置在哪 | 10–20 篇核心文献笔记 | ⬜ 待做 |
| ③ 立假设 | 把问题变成"可证伪"的假设 | "如果…那么…会提升X" | ⬜ |
| ④ 设计实验 | 想清楚要哪些表格/对比/消融 | 空表格(列名先定好) | ⬜ |
| ⑤ 做实验 | 跑、记录、迭代 | 填满的结果表 | ⬜ |
| ⑥ 写作 | 按特定顺序成文 | 4 页初稿 | ⬜ |
| ⑦ 打磨投稿 | 改、找人看、对格式、提交 | 最终 PDF | ⬜ |

下面逐个展开。

---

## 3. 阶段①：找一个"刚刚好"的问题

好问题的三个标准：**具体、可验证、有人关心**。

- ❌ 太大："让 CoT-TTS 更好" → 没法验证。
- ✅ 刚好："衡量并提升 CoT 推理与合成语音的情绪一致性" → 具体、能量化、契合挑战痛点。

**怎么找？** 三个来源：
1. **baseline 的缺口**：它哪里做得糙？（如重排只用了 mel 相似度、没人量化"言行一致"）
2. **挑战赛的评测维度**：官方在乎什么就往哪打（一致性占评分大头）。
3. **你跑实验时的意外**：经常最好的点子来自"咦这里怎么不对"。

> **给你的建议**：就用《ICASSP备战指南》里的**选题 A（一致性度量与重排）或 B（CoT 消融）**。现在就二选一，别再纠结——选错的代价远小于不开始的代价。

**产出**：写下一句话。例：
> "我们提出一个客观指标量化 CoT-TTS 中推理与语音的一致性，并用它做候选重排，在 X 数据上把一致性提升 Y%。"

---

## 4. 阶段②：读文献（新人最容易跳过、但最关键）

目的不是"读很多"，而是**搞清三件事**：别人怎么做的、用什么指标评、你和他们差在哪。

**高效读文献法**：
1. **先广后深**：用 Google Scholar / arXiv / Semantic Scholar 搜关键词（CoT-TTS、context-aware TTS、expressive/controllable TTS、LLM-based TTS、neural audio codec、speech emotion consistency）。
2. **滚雪球**：找到 1–2 篇最相关的，看它引用了谁、又被谁引用。
3. **三遍读法**：① 只读标题/摘要/图表，判断相不相关；② 读 Intro 和方法图，懂它做了啥；③ 真正相关的才精读。
4. **必读这一脉**：VALL-E、SparkTTS、CosyVoice、EnCodec/SoundStream（codec）、以及"语音情绪/风格评测"相关。
5. **做笔记**：每篇记三行——解决什么问题 / 用什么方法 / 用什么指标。这些笔记直接变成你的 Related Work。

**工具**：用 **Zotero**（免费）管理文献和自动生成 BibTeX 引用，别手动复制。

> 红线提醒：读文献时**留意有没有人已经做了你想做的**。如果有，看你能不能做得不同/更好；完全撞车就换角度。

---

## 5. 阶段③：把问题变成可证伪的假设

研究 = 提假设 + 用实验检验。把你的一句话问题改写成"如果…那么…"：

> **H1**：如果用情绪分类器比对 CoT 声称的情绪与合成语音的情绪作为重排分数，那么相比 mel 相似度重排，能提升语音-推理一致性指标，且不损害说话人相似度。

好处：假设逼你**提前想清楚要测什么、和谁比、怎么算赢**。

---

## 6. 阶段④：设计实验（先画空表格！）

**新手秘诀：论文的表格，在做实验之前就把"列名"定好。** 这叫"以终为始"。

先画出你最终想要的两张表（值留空）：

**主结果表（示意）**
| 方法 | 说话人相似度↑ | WER↓ | 情绪一致性↑ | MOS↑ |
|---|---|---|---|---|
| Baseline (mel rerank) | | | | |
| **Ours (consistency rerank)** | | | | |

**消融表（示意）**
| 配置 | 一致性↑ | … |
|---|---|---|
| 无 CoT | | |
| 完整 CoT | | |
| 去掉 Emotion 字段 | | |

**为什么先画表**：表格的每一个空格，就是你"必须跑的一个实验"。表格定了，实验清单就定了，不会东打一枪西打一枪。

**设计要点**：
- **公平对比**：baseline 和你的方法用**同数据、同评测、同超参基线**，只改你研究的那一个变量。
- **消融**：证明"是你那个改动起了作用"，而不是别的因素。
- **指标要全也要准**：客观（说话人相似度/WER/你的一致性指标/RTF）+ 小规模主观（MOS/CMOS）。中英双语都报。

---

## 7. 阶段⑤：做实验（记录比跑更重要）

新人最大的痛：**跑了一堆实验，最后忘了哪个配置出了哪个结果。** 防止方法：

1. **每个实验一个文件夹/一行记录**：配置（超参、checkpoint、随机种子）+ 结果数字 + 命令，都存下来。用一个 `experiments.md` 或表格记流水账。
2. **固定随机种子**（代码里已有 `--seed`），保证可复现。
3. **先小后大**：先在 5–10 条样例上跑通整条评测链，确认指标会动、方向对，再放大。
4. **尽早验证"有没有信号"**：两周内必须看到"我的方法让指标确实变好"。**没信号就果断换选题**，别耗。
5. **存好中间产物**：生成的 wav、CoT、meta.json 都留着，写作时要从里面挑例子、画图。

> 心态：**80% 的实验是为了证明一句话。** 不要被"再多跑一个配置"无限拖延。够支撑结论就停。

---

## 8. 阶段⑥：写作（按这个顺序写，最省力）

**千万别从 Introduction 开始写。** 推荐顺序：

```
1. 先把"主结果表 + 消融表 + 图"做好（你的实验产出）
2. Method —— 你最熟，照着代码/公式写
3. Experiments —— 描述数据/指标/实现，解读表格
4. Related Work —— 把文献笔记整理成段
5. Introduction —— 全文写完才最知道怎么"卖"
6. Abstract —— 最后写，150–200 词浓缩全文
7. Conclusion —— 收尾
8. Title —— 紧扣贡献，最后定
```

**各节怎么写（新手版要点）**：

- **Method**：让读者能复现。给公式、给算法步骤、给你重排分数的确切定义。先一句话交代 baseline（站在它肩上），重点写你新增的部分。
- **Experiments**：每张表配 2–4 句解读——"表 X 显示，我们的方法在一致性上从 A 提到 B，同时说话人相似度不降，说明…"。**数字要会说话**。
- **Introduction**（最难，占成败一半）：四段式套路——
  1. 大背景为什么重要；
  2. 现状/baseline 的**具体缺口**；
  3. 我们提出什么（一句话方法）；
  4. **贡献点 bullet**（2–3 条）+ 主结果数字。
- **Abstract**：问题→方法→关键结果数字，三句话能讲完。

**写作纪律**：
- 贡献点在 Abstract、Intro 结尾、结果表里**口径完全一致**地出现三次。
- 一段话一个中心意思；先结论后展开。
- 图表自包含（看标题和图例就懂，不用翻正文）。
- 用 **Overleaf + IEEEtran 模板**（在线协作、不用配环境）。

## 9. 阶段⑦：打磨与投稿

1. **自己冷读一遍**：放一天再读，站在审稿人角度问"我凭什么信你？"
2. **找人看**：导师/师兄师姐/同行至少一轮。**别人看 10 分钟发现的问题，比你自己看 10 小时多。**
3. **对照投稿前清单**（见《ICASSP备战指南》第 7 节）：原创性、主结果+消融、公平对比、双语、引用致谢、4 页格式。
4. **查重 & 引用规范**：所有借用的代码/模型/数据（SparkTTS、VeOmni、挑战赛数据）都要引用+致谢；注意 dual-submission 红线。
5. **提前 1–2 天提交**：截止前系统常崩。ICASSP 2027 截止 **2026-09-16 (AoE)**。

## 10. 新人最常踩的 10 个坑

1. 选题太大，做不完也讲不清 → 越具体越好。
2. 不读文献，重复造轮子或撞车。
3. 不记录实验，结果对不上配置。
4. 对比不公平（偷偷给自己方法调更多参）→ 审稿人一眼看穿。
5. 只报一个对自己有利的指标 → 要全面、诚实。
6. 把"我做了什么"写成流水账，没有论证逻辑。
7. 图表丑、看不懂、和正文口径不一。
8. 拖到最后才写作，发现缺实验来不及补。
9. 不留时间给反馈和润色。
10. 卡点提交，系统崩了错过截止。

## 12. 把它接到你当前的进度（行动清单）

你已经完成"理解系统"，现在缺的就是"做出并写出一个贡献"。本周可执行：

- [ ] **今天**：在选题 A / B 中二选一，写下你的一句话研究问题。
- [ ] **本周**：读 8–10 篇核心文献（VALL-E、SparkTTS、CosyVoice、codec、语音情绪评测），各记三行笔记。
- [ ] **本周**：画出你的主结果表 + 消融表的"空表格"。
- [ ] **下周**：搭好评测脚本（说话人相似度/WER/情绪分类器），在 5–10 条样例上跑通。
- [ ] **两周内**：验证"我的方法让指标变好"——有信号就推进，没信号就换 A↔B。
- [ ] 之后按第 5–9 阶段推进，对齐《ICASSP备战指南》的 11 周时间表。

## 13. 最后，给新人的几句真心话

- **完成 > 完美。** 第一篇论文的最大价值是"走完整个流程"，哪怕中了普通会议、哪怕被拒后改投——你学到的流程会跟你一辈子。
- **被拒很正常。** ICASSP 录用率不高，顶级研究者也常被拒。审稿意见是免费的改进建议，按它改、再投。
- **找一个 mentor。** 有经验的人点一句，胜过你自己摸索一周。主动找导师/师兄聊你的选题。
- **早写、多给人看。** 这两件事的回报率最高。

> 一句话总结：**锁定一个小问题 → 读懂别人 → 先画好结果表 → 做实验验证 → 按"先方法后引言"的顺序写 → 找人改 → 提前提交。** 你已经站在系统理解的肩膀上了，剩下的就是把这条流程走一遍。

需要的话，我可以帮你把**选题 A 或 B 细化成具体实验方案 + 评测脚本设计**，或起一个 **Overleaf LaTeX 论文骨架**，让你直接往里填。
