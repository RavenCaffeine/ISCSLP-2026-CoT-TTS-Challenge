# 把这套 CoT-TTS 工作投 ICASSP —— 备战指南

> 基于 ICASSP 2027 官方 Call for Papers（已核实）+ 本仓库 CoT-TTS 工作的现状
> 配套：《架构与运作流程解析》《Track1 任务文档》《简历与面试准备》

---

## 0. 先回答：这事能不能成？

**能，但有一个硬门槛**：ICASSP 要的是**原创、未发表的技术论文**。只"跑通官方 baseline + 分析"**不够**——那是工程复现，不是研究贡献。你必须在 baseline 之上做出**一个有新意、有实验支撑的点**。好消息是：你已经把整套系统吃透了，离"做出一个改进点"只差一步。本指南就是教你把这一步走完。

⚠️ **两条红线先记住（见第 6 节详述）**：
1. **原创性**：核心方法/发现必须是你的，不能把官方 baseline 包装成自己的系统。
2. **重复投稿**：如果你打算把同一套结果投 ISCSLP 2026 Challenge（或其 special session），**就不能再把同样内容投 ICASSP**——这违反 IEEE 的 dual-submission 规定。要么两边内容有实质差异，要么只投一个。

## 2. ICASSP 论文是什么样的（先建立预期）

- **篇幅**：通常**4 页正文 + 参考文献**（第 5 页一般仅放参考文献）。务必下载官方 **Author Kit / 模板**确认当年的确切规定。
- **格式**：IEEE 双栏模板（LaTeX 或 Word，强烈建议 LaTeX + Overleaf 的 IEEEtran 模板）。
- **评审**：单盲（评审看得到作者，但你看不到评审）——所以**不需要匿名化**，但也别指望"挑战赛名气"加分，论文靠内容说话。
- **气质**：ICASSP 偏**信号处理/方法**。一篇典型录用论文 = 一个清晰的问题 + 一个具体方法/改进 + 扎实的客观实验（指标表 + 消融）+ 诚实的对比。**4 页放不下长篇大论，贵在聚焦。**

## 3. 选题：从 baseline 到"一个可发表的贡献"

你不需要发明全新系统。ICASSP 欢迎**针对具体问题的、可量化验证的改进或分析**。下面按"投入产出比"排序，**选其中一个做深**即可：

### 🟢 推荐档（4 页友好、2.5 个月可完成）

| # | 选题方向 | 核心贡献点 | 需要的实验 |
|---|---|---|---|
| **A** | **语音-推理一致性度量与优化** | 提出一个量化"CoT 推理 ↔ 合成语音"是否一致的指标（如用情绪分类器/SV 模型比对 CoT 声称的情绪与实际语音情绪），并据此做候选重排，证明能提升一致性 | 在 baseline 上加你的重排，对比一致性指标 + 主观/客观分 |
| **B** | **CoT 的消融研究（Ablation）** | 系统性回答"CoT 到底有没有用、哪些字段有用"——有/无 CoT、打乱 CoT、各字段逐一去除对表现力与语境贴合的影响 | 多组配置对比，客观指标(说话人相似度/WER/情绪准确率) + 小规模主观 |
| **C** | **重排策略改进** | 把 baseline 里朴素的 `mel_similarity` 重排，换成说话人验证(SV)打分 + CoT 一致性打分的组合，证明音色相似度/表现力双提升 | 不同重排策略的对比表 + 消融 |

### 🟡 进阶档（更亮眼但更费时，团队/有 GPU 资源更稳）

| # | 选题方向 | 贡献点 |
|---|---|---|
| **D** | **轻量微调对比** | 不同 LoRA rank / target 模块 / 采样策略对表现力与稳定性的影响，给出实用配置建议 |
| **E** | **推理加速** | 针对自回归 TTS 慢的问题，做投机解码/缓存/截断策略，报告 RTF(实时率) 与质量的权衡 |
| **F** | **失败模式分析 + 修复** | 系统刻画 baseline 的失败（漏字、情绪不符、音色漂移），提出针对性修复并验证 |

> **给个人/时间紧者的明确建议：选 A 或 B。** 两者都能在现有 baseline 上做，不需要重训大模型，实验量可控，且贡献点("一致性"正好是这个挑战的核心痛点)很契合 ICASSP 的口味。

## 4. 论文结构模板（4 页，照着填）

```
Title: 紧扣贡献，例如
  "Measuring and Improving Speech–Reasoning Consistency in Chain-of-Thought TTS"

Abstract (~150–200 词)：问题 + 方法 + 主要结果数字

1. Introduction（~0.75 页）
   - CoT-TTS / 语境感知表现力合成的背景与意义
   - 指出现有 baseline 的具体缺口（如：CoT 与语音是否真的一致，无人量化）
   - 用 2–3 个 bullet 列出你的贡献(contributions)

2. Related Work（~0.5 页）
   - LLM-based TTS（VALL-E、SparkTTS、CosyVoice 等）
   - 神经音频编解码（EnCodec/SoundStream/BiCodec）
   - 可控/表现力 TTS、CoT 在语音中的应用

3. Method（~1 页，核心）
   - 简述 baseline 流程（你站在它肩上，要交代清楚但别长篇）
   - 重点讲【你的方法】：指标定义/重排公式/消融设计，给清楚公式和算法

4. Experiments（~1.25 页，决定能否录用）
   - 数据集(HKUSTAudio/chall)、评测指标、实现细节(超参/GPU)
   - 主结果表 + 消融表 + 必要的图
   - 客观指标(说话人相似度、WER、情绪准确率、RTF…) + 小规模主观(MOS/CMOS)

5. Conclusion（~0.25 页）

References（第 5 页）
```

**写作铁律**：贡献点在 Abstract 第 2 句、Intro 末尾 bullet、和结果表里**各出现一次且口径一致**。审稿人 10 分钟决定命运，别让他猜你做了什么。

## 5. 倒排时间表（从 9/16 往回推）

| 周次 | 时间 | 任务 |
|---|---|---|
| W1 | 6 月底–7 月初 | **定选题(选 A 或 B)** + 跑通 baseline + 搭好评测脚本(SV/WER/情绪分类器) |
| W2–W3 | 7 月 | 实现你的方法/消融配置，跑初步实验，确认"有信号"(指标真的动了) |
| W4–W5 | 7 月底–8 月初 | 补全实验：主结果 + 消融 + 多语言(中英)；整理数据表 |
| W6 | 8 月中 | 搭小规模主观评测(找几个人做 MOS/CMOS，哪怕 10–20 人 × 20 句) |
| W7–W8 | 8 月中下 | **写初稿**(先写 Method+Experiments，再写 Intro/Related) |
| W9 | 9 月初 | 找导师/同行**至少过一轮反馈**，补漏实验 |
| W10 | 9 月 8–14 | 打磨图表、润色英文、查重、对齐格式、压到 4 页 |
| W11 | 9 月 15–16 | **提前一天提交**(系统常因截止前拥堵崩溃，别卡点) |

> **最大风险是"实验没信号"**：所以 W1–W3 必须尽快验证"你的改进确实让指标变好"。如果两周内完全没信号，果断换选题(A↔B)，别在死胡同耗到 8 月。

## 6. 三个必须避开的坑（合规 & 学术诚信）

1. **原创性**：论文的卖点必须是你新增的方法/发现。Baseline 要明确**引用并标注**为他人工作（SparkTTS/VeOmni/挑战赛官方），不能含糊其辞让人以为是你的。
2. **重复投稿(dual submission)**：IEEE 严禁同一工作同时投多处。如果你也参加 ISCSLP 2026 Challenge 并写 system/challenge paper，**ICASSP 的稿件必须有实质性不同**（不同方法、不同核心结果），否则二选一。拿不准就问导师或挑战赛组织方。
3. **数据与许可**：HKUSTAudio/chall 数据集、Spark-TTS(Apache-2.0)、VeOmni 等都要按其 license 使用并在论文中致谢/引用；数据来自影视/广播剧，注意论文里不要泄露受版权保护的内容样例。

## 7. 投稿前自检清单

- [ ] 贡献点是否原创、清晰、可一句话说出？
- [ ] 有没有至少一张**主结果表**和一张**消融表**，数字支撑结论？
- [ ] baseline 与对比方法是否在**同等条件**下比较（同数据、同评测）？
- [ ] 客观指标 + 至少小规模主观，结论不靠单一指标？
- [ ] 中英双语都报告了吗（挑战赛是双语）？
- [ ] 所有借用的代码/模型/数据是否正确引用与致谢？
- [ ] 是否与你可能投的 ISCSLP challenge 稿件构成重复投稿？
- [ ] 格式压到 4 页 + 参考文献页，用官方 IEEE 模板？
- [ ] 找人读过、英文与图表清晰？
- [ ] 提前 1–2 天提交了？
